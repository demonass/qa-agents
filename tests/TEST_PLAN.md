## 2. 测试目标

本测试计划旨在验证 **QA Agent**（基于 ReAct + Skills 架构的智能测试助手）的功能正确性、性能表现和安全性，确保系统达到生产环境要求。

### 2.1 测试范围

| 测试层级 | 覆盖范围 | 优先级 |
|----------|----------|--------|
| **单元测试** | ReAct Agent 核心逻辑、技能路由器、工具管理器 | 高 |
| **集成测试** | Agent 与技能、工具、缓存的交互 | 高 |
| **系统测试** | 端到端用户流程、多轮对话 | 高 |
| **LLM评测** | 忠实度、相关性、幻觉检测 | 中 |
| **性能测试** | TTFT、TPS、并发处理 | 中 |
| **安全测试** | 提示词注入、路径遍历、权限控制 | 高 |

### 2.2 质量门禁

| 指标 | 目标值 |
|------|--------|
| 单元测试覆盖率 | ≥ 70% |
| 集成测试通过率 | 100% |
| LLM忠实度 | ≥ 0.7 |
| LLM答案相关性 | ≥ 0.7 |
| TTFT（并发） | < 2秒 |
| TPS | > 20 tokens/s |
| 缓存命中率 | > 60% |
| 错误率 | < 1% |

## 5. 测试执行策略

### 5.1 CI/CD双轨制

```yaml
# 提交触发（快速测试 - 0成本）
push/pull_request:
  - 单元测试（Mock掉外部API）
  - 集成测试
  - LLM测试（使用Mock）
  - 代码质量检查

# 定时触发（完整质量评测）
schedule:
  - 完整测试套件
  - 性能测试
  - 安全测试
  - 覆盖率报告
```

### 5.2 测试执行命令

```bash
# 运行所有单元测试
python3 -m pytest tests/unit/ -v --cov=agents --cov-report=term

# 运行集成测试
python3 -m pytest tests/integration/ -v

# 运行LLM评测（需要安装Ragas/DeepEval）
python3 -m pytest tests/llm/ -v

# 运行性能测试（带--run-slow标志）
python3 -m pytest tests/performance/ -v --run-slow

# 运行完整测试套件
python3 -m pytest tests/ -v --cov=agents --cov=skills --cov=tools
```

### 5.3 测试优先级

| 优先级 | 测试类型 | 执行频率 |
|--------|----------|----------|
| P0 | 单元测试 | 每次提交 |
| P0 | 集成测试 | 每次提交 |
| P1 | LLM评测 | 每日/发布前 |
| P1 | 安全测试 | 每日/发布前 |
| P2 | 性能测试 | 每周/发布前 |

## 7. 风险与缓解措施

| 风险 | 描述 | 影响 | 缓解措施 |
|------|------|------|----------|
| LLM API成本 | 测试调用真实API产生费用 | 高 | 使用Mock或本地模型 |
| 测试不稳定 | 网络波动导致CI失败 | 中 | 设置重试机制 |
| 幻觉测试误报 | 模型随机性导致测试失败 | 中 | 使用量化指标，设置合理阈值 |
| 性能测试差异 | 环境差异导致结果不一致 | 低 | 使用标准化测试环境 |
